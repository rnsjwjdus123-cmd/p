## 3. 매칭 알고리즘

### 점수 계산

```
Q4~Q14 (11문항): 주 향 +3, 부 향 +1
Q15 (1문항): 주 향 +4.5, 부 향 +1.5 (x1.5 가중치)

최대 가능 점수 (한 향에 올인):
  11 × 3 + 4.5 = 37.5점
```

### 향 성격 → 제품 매칭 테이블

**Q3에서 "Living room or bedroom" 선택 시:**

| 순위 향 성격 | 추천 제품 |
|------------|----------|
| FLORAL (1위) | 왕실의 모란 (캔들침실) |
| FLORAL (2위) | 나주 배꽃의 향연 (캔들침실) |
| GREEN (1위) | 남산의 새벽 안개 (룸스프레이) |
| GREEN (2위) | 기와의 기억 (룸스프레이) |
| FRESH | 제주 소금과 민트 (룸스프레이) |
| WOODY_INK | 단청 아래 빗소리 (캔들침실) |
| WARMING | 고가구의 온기 (캔들 — 침실 겸용 추천) |

**Q3에서 "Kitchen or dining" 선택 시:**

| 순위 향 성격 | 추천 제품 |
|------------|----------|
| FRESH / FLORAL | 생강과 유자 |
| WOODY_INK / GREEN | 조용한 찻자리 |
| WARMING | 고가구의 온기 |

> 주방 캔들은 3종이므로 점수 1위 향 성격 기준으로 가장 가까운 제품 매칭.

**Q3에서 "Car" 선택 시:**

| 순위 향 성격 | 추천 제품 |
|------------|----------|
| GREEN / FRESH | 흑송과 호박 (자연적 + 상쾌한) |
| WOODY_INK (1위) | 한지 위 먹향 |
| WOODY_INK (2위) / WARMING | 묵향의 자취 |

### 동점 해소 규칙

```
1. Q15 점수가 높은 향 우선
2. 그래도 동점 → Q14 → Q12 순서 역추적
3. 최종 동점 → 카테고리 내 첫 번째 제품 기본값
```

## 5. 기술 흐름

```
[퀴즈 페이지 HTML — 자사몰 디자인 적용]
  ↓ 유저가 15문항 응답
  ↓ JS에서 점수 계산 → 향 성격 결정 → Q3 공간 + 향 성격 교차 → 제품 매칭
  ↓
  ├→ Firebase Firestore에 전체 결과 저장 (데이터 분석용)
  └→ n8n Webhook POST (subscriber_id, matched_product, scores)
       ↓
       n8n → DM 메시지 생성 → ManyChat API → Instagram DM 발송
```

## 6. 주의사항 및 이슈

### ⚠️ 밸런스 이슈
- **WARMING** 향 성격에 해당하는 제품이 "고가구의 온기" 1종뿐
- **FRESH** 향 성격도 "제주 소금과 민트" + "생강과 유자" 2종
- 반면 **WOODY_INK**는 5종으로 과다
- → 추후 제품 추가 시 WARMING, FRESH 라인업 보강 권장

### ⚠️ 디퓨저 카테고리 미존재
- 원래 4제품(디퓨저, 차량용, 캔들, 룸스프레이) 중 "디퓨저" 해당 제품이 리스트에 없음
- → Q3 선택지에서 디퓨저 제외하고 공간 기반으로 변경함

### ✅ 클린 코튼 처리
- "클린 코튼(의문...)" → "FRESH" 카테고리에 통합
- 제주 소금과 민트가 클린/프레시 느낌을 커버
