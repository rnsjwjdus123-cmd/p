# Strategy 분석 — 어디서 오래 걸리는지 (확인용)

수정 없이 **흐름만** 정리한 문서입니다.

---

## 실행 순서 (server.py 기준)

| 순서 | 단계 | 하는 일 | 예상 구간 |
|------|------|----------|-----------|
| **1** | Perplexity 인플루언서 검색 | `search_perplexity()` 1회 (인플루언서 검색) | ~10–30초 (API timeout 30초) |
| **2** | OpenAI 메인 분석 | `analyze_with_openai()` 1회 — **6탭 한 번에** (타겟 페르소나, UK 적합성, 셀링포인트, 인플루언서 전략, 연락 전략 등) | **~30–90초** (프롬프트 크고, timeout 120초) |
| **3** | Phase 1 (병렬) | Perplexity **경쟁사** 검색 (`_search_comp`) + **해시태그** 검색 (`_search_hash`) | 각각 앞에 **time.sleep(1), time.sleep(2)** 있음 → 최소 2초 + 검색 각 ~10–30초. use_competitor만 켜면 경쟁사 1개만 실행 (1초 sleep + 1회 Perplexity) |
| **4** | Phase 2 | ROI 시뮬레이션 계산 | 로컬 계산만 → **거의 즉시** |
| **5** | Phase 3 (병렬) | GPT 분석 4종 중 요청된 것만: 경쟁사 분석, ROI 인사이트, 타임라인, 해시태그 트렌드 | 각 **timeout 120초**. use_competitor + use_roi만 켜면 **2개 동시** (경쟁사 GPT, ROI 인사이트) → 둘 중 더 오래 걸리는 쪽만큼 대기 |

---

## 프론트에서 보내는 옵션 (strategy.html)

- `use_perplexity: true`
- `use_competitor: true`
- `use_roi: true`
- `use_timeline: false`
- `use_hashtag: false`

→ **실제로 도는 것:**  
1번 Perplexity(인플루언서) → 2번 OpenAI 메인(6탭) → 3번 Perplexity(경쟁사 1개, 1초 sleep 포함) → 4번 ROI 로컬 → 5번 GPT 2개 병렬(경쟁사, ROI 인사이트).

---

## 오래 걸릴 수 있는 구간 (추정)

| 구간 | 이유 |
|------|------|
| **2. OpenAI 메인 분석** | 한 번에 6탭 분량, 프롬프트 크고 토큰 많음. OpenAI 응답이 30초~90초 넘어갈 수 있음. |
| **5. Phase 3 GPT 2개** | 경쟁사 분석·ROI 인사이트 각각 별도 OpenAI 호출. 병렬이라도 둘 다 끝날 때까지 대기. Render 무료 tier에서 CPU/네트워크 느리면 각각 1분 넘게 걸릴 수 있음. |
| **1, 3 Perplexity** | 각 10–30초 구간. 네트워크/API 지연 시 더 늘어날 수 있음. |
| **Render 무료 인스턴스** | 슬립 후 깨우기 + CPU/메모리 제한으로 **모든 단계가 2~3배 느려질 수 있음**. |

---

## 10분 걸렸을 때 가능성

- **2번(OpenAI 메인)** 이 2분 넘게 걸리거나,  
- **5번(GPT 2개)** 중 하나가 120초 근처까지 걸리거나,  
- Render 쪽에서 **전체가 throttling** 되어 1·2·3·5가 모두 느려진 경우  

→ 합쳐서 **5~10분**까지 나올 수 있음.

## 다음에 할 수 있는 것 (참고만)

- **로그 추가**: 각 단계 **시작/끝 시각**을 `steps`에 넣어서, 실제로 1·2·3·5 중 어디가 가장 오래 걸리는지 측정.
- **병렬화**: 1번 Perplexity와 2번 OpenAI 메인은 서로 독립이면 동시에 돌리기 (현재는 1 → 2 순서).
- **Phase 1 sleep 제거**: `time.sleep(1)`, `time.sleep(2)` 는 의도된 지연이 아니라면 제거 검토.
- **Render 유료**: CPU/메모리 여유 생기면 단계별 체감 시간 단축 가능.

이 문서는 **확인용**이며, 코드 수정은 하지 않았습니다.
