# Santokki (p) 배포 가이드

배포 구성을 한눈에 보고, 순서대로 진행할 수 있는 체크리스트입니다.

---

## 배포 구성 요약

| 대상 | 서비스 | 내용 |
|------|--------|------|
| **쇼핑몰·Hub·Strategy·Dashboard 페이지** | **Netlify** | 정적 HTML/CSS/JS. API 없음. |
| **프롬프트 생성 API** (생성하기 버튼, Firebase 통계, 구글 시트) | **Render** (선택) | Node(server.js). 없으면 해당 기능은 로컬에서만 동작. |
| **시향 퀴즈·대시보드** | **Firebase Hosting** | santokki/quiz, santokki/dashboard. 이미 배포된 경우 그대로 사용. |
| **Strategy/Dashboard 분석 API** (경쟁사·ROI 등) | 로컬 또는 별도 서버 | project_html 백엔드(Python). 보통 로컬 `http://localhost:8000` 사용. |

---

## 1. Netlify — 정적 사이트 (필수)

쇼핑몰, hub.html, strategy.html, unified-dashboard.html, prompt-generator.html **페이지**가 열리는 곳입니다. (버튼 동작은 API 배포 후 가능)

### 1.1 체크

- [ ] Netlify 계정 있음 ([netlify.com](https://netlify.com))
- [ ] GitHub에 `p` 저장소 푸시됨

### 1.2 배포 절차

1. Netlify **Dashboard** → **Add new site** → **Import an existing project**
2. **GitHub** 연결 후 저장소 `p` 선택
3. 설정 확인:
   - **Build command:** 비움 (또는 없음)
   - **Publish directory:** `.` (루트 그대로)
   - **Branch:** `main`
4. **Deploy site** 클릭

### 1.3 배포 후

- 사이트 URL: `https://랜덤이름.netlify.app`
- 쇼핑몰: `https://랜덤이름.netlify.app/`
- Hub: `https://랜덤이름.netlify.app/hub.html`
- 대시보드: `https://랜덤이름.netlify.app/santokki/dashboard.html`

**참고:** `netlify.toml`, `_redirects` 가 있어서 `/santokki/dashboard` → `dashboard.html` 리다이렉트는 자동 적용됩니다.

## 2. Render — Node API (선택, 프롬프트 생성용)

**「생성하기」**, **Firebase 통계 자동 생성**, **구글 시트 저장**을 Netlify 밖(또는 로컬 없이) 쓰려면 필요합니다.

- 상세: **[docs/배포_프롬프트생성기.md](배포_프롬프트생성기.md)** 참고
- 요약:
  1. Render에서 **Web Service** 생성, 저장소 연결
  2. **Build:** `npm install` / **Start:** `npm run start:api`
  3. 환경 변수: `OPENAI_API_KEY` (필수), `LINKTREE_URL`, `FIREBASE_SERVICE_ACCOUNT_JSON`, `GOOGLE_SHEET_ID` 등 (선택)
  4. 배포 후 URL 예: `https://서비스이름.onrender.com`

- **프롬프트 생성기 페이지를 Netlify에만 올리면** 「생성하기」는 동작하지 않습니다 (API가 없음).
- **Render에 배포하면** 같은 도메인에서 페이지 + API를 제공하므로, `https://서비스이름.onrender.com/prompt-generator.html` 로 접속해 생성 기능을 사용할 수 있습니다.

## 3. Firebase Hosting — 퀴즈·대시보드 (이미 있는 경우)

- 퀴즈: `santokki/quiz/` → `firebase deploy --only hosting` (santokki 폴더 또는 quiz 폴더에서)
- 대시보드: 동일 프로젝트에 포함하거나 별도 설정
- 자세한 절차: **santokki/README.md** § 퀴즈 페이지 배포 참고

---

## 4. Strategy / Dashboard 분석 API (project_html)

- **용도:** hub → Strategy, Dashboard 탭의 「Run strategy analysis」 등 (Python FastAPI, 포트 8000)
- **기본:** 로컬에서 `cd project_html/backend && uvicorn server:app --port 8000` 실행 후 사용
- **배포:** Render(Runtime: Python), Railway, Fly.io 등에 별도 배포 가능. 배포 시 루트 `.env` 또는 backend `.env`에 `OPENAI_API_KEY`, `PERPLEXITY_API_KEY` 설정 필요.

## 5. 배포 전 체크리스트

- [ ] `.env`, `firebase-key.json`, `tidy-drive*.json` 등 **Git에 올리지 않음** (`.gitignore` 확인)
- [ ] `npm install` 후 `npm run start` 로 로컬에서 페이지 열림 확인
- [ ] (API 쓸 경우) `npm run start:api` 로 API 동작 확인
- [ ] Netlify 배포 후 메인·hub·대시보드 URL 접속 테스트
