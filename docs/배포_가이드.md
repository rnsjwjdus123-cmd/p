# Santokki (p) 배포 가이드

배포 구성을 한눈에 보고, 순서대로 진행할 수 있는 체크리스트입니다.

---

## 배포 구성 요약

| 대상 | 서비스 | 내용 |
|------|--------|------|
| **쇼핑몰·Hub·Strategy·Dashboard 페이지** | **Netlify** | 정적 HTML/CSS/JS. API 없음. |
| **프롬프트 생성 API** (생성하기 버튼, Firebase 통계, 구글 시트) | **Render** (선택) | Node(server.js). 없으면 해당 기능은 로컬에서만 동작. |
| **시향 퀴즈·대시보드** | **Firebase Hosting** | santokki/quiz, santokki/dashboard. 이미 배포된 경우 그대로 사용. |
| **Strategy/Dashboard 분석 API** (경쟁사·ROI 등) | 로컬 또는 별도 서버 | project_html 백엔드(Python). 보통 로컬 `http://localhost:8000` 사용. |

## 1. Netlify — 정적 사이트 (필수)

쇼핑몰, hub.html, strategy.html, unified-dashboard.html, prompt-generator.html **페이지**가 열리는 곳입니다. (버튼 동작은 API 배포 후 가능)

### 1.1 체크

- [ ] Netlify 계정 있음 ([netlify.com](https://netlify.com))
- [ ] GitHub에 `p` 저장소 푸시됨

### 1.2 배포 절차

1. Netlify **Dashboard** → **Add new site** → **Import an existing project**
2. **GitHub** 연결 후 저장소 `p` 선택
3. 설정 확인:
   - **Build command:** 비움 (또는 없음)
   - **Publish directory:** `.` (루트 그대로)
   - **Branch:** `main`
4. **Deploy site** 클릭

### 1.3 배포 후

- 사이트 URL: `https://랜덤이름.netlify.app`
- 쇼핑몰: `https://랜덤이름.netlify.app/`
- Hub: `https://랜덤이름.netlify.app/hub.html`
- 대시보드: `https://랜덤이름.netlify.app/santokki/dashboard.html`

**참고:** `netlify.toml`, `_redirects` 가 있어서 `/santokki/dashboard` → `dashboard.html` 리다이렉트는 자동 적용됩니다.

## 2. Render — Node API (프롬프트 생성·구글 시트 등)

**「생성하기」**, **Firebase 통계 자동 생성**, **구글 시트 저장**, **Hub 구글 시트 링크**를 배포에서 쓰려면 필요합니다.

- 상세: **[docs/배포_프롬프트생성기.md](배포_프롬프트생성기.md)** 참고
- 요약:
  1. Render에서 **Web Service** 생성, 저장소 연결
  2. **Build:** `npm install` / **Start:** `npm run start:api`
  3. 환경 변수: `OPENAI_API_KEY` (필수), `LINKTREE_URL`, `FIREBASE_SERVICE_ACCOUNT_JSON`, `GOOGLE_SHEET_ID` 등 (선택)
  4. 배포 후 URL 복사 (예: `https://서비스이름.onrender.com`)
  5. **저장소 루트의 `config.json`** 수정:
     ```json
     { "apiBase": "https://서비스이름.onrender.com" }
     ```
     위처럼 Render URL을 넣고 커밋·푸시하면 Netlify가 다시 배포되면서, Netlify에 올라간 프롬프트 생성기·Hub가 이 API 주소를 사용합니다.

- **프롬프트 생성기 페이지를 Netlify에만 올리면** 「생성하기」는 API가 없어 동작하지 않습니다.
- **Netlify + Render + config.json 설정**까지 하면, Netlify 주소에서 열린 프롬프트 생성기·Hub가 Render API를 불러서 기능이 동작합니다.

### 2.1 「Firebase 연동 없음」 오류 해결

**증상:** 프롬프트 생성기에서 "Firebase 통계로 자동 생성" 버튼 아래에 **"Firebase 연동 없음. santokki/firebase-key.json 확인."** 빨간 메시지가 뜸.

| 환경 | 해결 방법 |
|------|-----------|
| **로컬** (`npm run start:api`) | `santokki/firebase-key.json` 파일이 있는지 확인. 없으면 [아래](#firebase-키-받는-방법)처럼 Firebase 콘솔에서 서비스 계정 JSON 키를 다운로드한 뒤 `santokki/firebase-key.json`으로 저장. |
| **Render 배포** | Render에는 `firebase-key.json`을 올리지 않으므로, **환경 변수**로 넣어야 함. Render Dashboard → 해당 Node API 서비스 → **Environment** → **Add Environment Variable** → 이름: `FIREBASE_SERVICE_ACCOUNT_JSON`, 값: `santokki/firebase-key.json` **전체 내용을 한 줄**로 붙여넣기 (줄바꿈 없이). 저장 후 **Manual Deploy** 한 번 실행. |

**Firebase 키 받는 방법 (로컬용 파일 / Render용 JSON 값):**

1. [Firebase Console](https://console.firebase.google.com) → 프로젝트 선택 (예: santokki-f7c72)
2. ⚙️ **프로젝트 설정** → **서비스 계정** 탭
3. **새 비공개 키 생성** → JSON 파일 다운로드
4. **로컬:** 다운로드한 파일 이름을 `firebase-key.json`으로 바꿔서 `p/santokki/` 폴더에 넣기
5. **Render:** 그 JSON 파일을 메모장 등으로 열어 **한 줄로** 복사(줄바꿈 제거) 후 `FIREBASE_SERVICE_ACCOUNT_JSON` 값에 붙여넣기

## 4. Strategy / Dashboard 분석 API (project_html)

Hub → **Strategy**, **Dashboard** 탭의 「Run strategy analysis」, 경쟁사·ROI 분석을 **배포 환경**에서 쓰려면 Python 백엔드를 별도 배포해야 합니다.

### 4.1 로컬만 쓸 때

- `cd project_html/backend && uvicorn server:app --port 8000` 실행
- strategy.html / unified-dashboard.html 에서 **Analysis API URL** 이 `http://localhost:8000` 이면 그대로 동작

### 4.2 Render에 Python 서비스로 배포할 때

1. Render **Dashboard** → **New** → **Web Service**
2. **같은 GitHub 저장소** `p` 선택
3. **Root Directory:** `project_html/backend` 로 지정 (또는 저장소 루트로 두고 Build 시 `cd project_html/backend` 포함)
4. **Runtime:** **Python**
5. **Build command:** `pip install -r requirements.txt` (Root가 `project_html/backend` 면) 또는 `cd project_html/backend && pip install -r requirements.txt`
6. **Start command:** `uvicorn server:app --host 0.0.0.0 --port $PORT`
7. **Environment:** `OPENAI_API_KEY`, `PERPLEXITY_API_KEY` (필수. 경쟁사·인플루언서 검색에 사용)
8. 배포 후 **서비스 URL** 복사 (예: `https://santokki-strategy.onrender.com`)
9. **config.json** 에 추가:
   ```json
   {
     "apiBase": "https://api-santokki-2835.onrender.com",
     "strategyApiBase": "https://santokki-strategy.onrender.com"
   }
   ```
   커밋·푸시하면 Netlify 재배포 시 strategy.html / unified-dashboard.html 이 이 주소를 자동으로 사용합니다.

**참고:** Root Directory를 `project_html/backend` 로 두면 Render가 그 폴더 기준으로 빌드·실행합니다. requirements.txt 는 `project_html/backend/requirements.txt` 에 있습니다.
